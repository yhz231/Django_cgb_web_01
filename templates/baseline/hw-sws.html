1.1 设备命名配置

设备命名需采用以下统一命名规则：

AA-BB-CC-DD-EE

  • AA: 分行标识
  • BB: 功能区域代码
  • CC: 设备类型代码
  • DD: 机柜位置信息(可选)
  • EE: 设备编码

设备命名配置：

hostname XXFH-CO-CS-01

1.3 AAA配置

AAA ，认证(Authentication)：验证用户的身份与可使用的网络服务；授权(Authorization)：依据认证结果开放网络服务给用户；计帐(Accounting)：记录用户对各种网络服务的用量，并提供给计费系统。整个系统在网络管理与安全问题中十分有效。常用的AAA协议是Radius（RFC 2865，RFC 2866），TACACS（RFC1492），华为设备AAA认证主要用到的是HWTACACA协议。

分行核心设备指向总行3A服务器，其他设备指向分行的3A服务器。

1.5 生成树配置

生成树协议STP(Spanning Tree Protocol)的主要功能有两个：一是在利用生成树算法、在以太网络中，创建一个以某台交换机的某个端口为根的生成树，避免环路。二是在以太网络拓扑发生变化时，通过生成树协议达到收敛保护的目的。分行使用的生成树协议有快速生成树RSTP（思科设备），与多进程生成树协议MSTP（非思科设备）

生成树部分配置说明：

表1-6                                                     生成树配置

配置命令 作用
stp enable 全局使能STP
stp instance 0 root primary 设置实例0优先级为 primary
stp mode mstp 配置生成树类型为MSTP
stp bpdu-protection 配置BPDU保护

修改生成树类型以及根节点会引起生成树重新收敛，可能导致业务流量中断，建议选择变更窗口期进行修改。

生成树配置命令如下：

主交换机，配置如下：
stp enable
#
stp instance 0 root primary
stp mode mstp
stp bpdu-protection
#
备交换机，配置如下：
stp enable
#
stp instance 0 root secondary
stp mode mstp
stp bpdu-protection
#
其它交换机，配置如下：
stp enable
#
stp mode mstp
stp bpdu-protection
1.6 SNMP配置

SNMP(Simple Network Management Protocol)是一种简单网络管理协议，它属于TCP/IP五层协议中的应用层协议，用于网络管理的协议。SNMP主要用于网络设备的管理。由于SNMP协议简单可靠 ，受到了众多厂商的欢迎，成为了目前最为广泛的网管协议。

SNMP的工作方式：管理员需要向设备获取数据，所以SNMP提供了【RO】操作；管理员需要向设备执行设置操作，所以SNMP提供了【RW】操作；设备需要在重要状况改变的时候，向管理员通报事件的发生，所以SNMP提供了【Trap】操作。

SNNP部分配置说明：

表1-7                                                     SNMP配置

配置命令 作用
acl name newwg 定义允许的SNMP服务器范围
rule 10 permit source 21.14.0.0 0.0.0.255 允许总行网管地址
rule 20 permit source 10.2.95.211 0 允许总行网管地址
snmp-agent 使能SNMP代理功能

snmp-agent community read zxcv1234 acl newwg 设置团体名隐藏，匹配newwg的源地址报文允许使用该团体名进行只读访问
snmp-agent community complexity-check disable 关闭团体名密码复杂度检查功能

snmp-agent community complexity-check disable

1.8 NTP配置

NTP服务器【Network Time Protocol（NTP）】是用来使计算机时间同步化的一种协议，它可以使计算机对其服务器或时钟源（如石英钟，GPS等等)做同步化，它可以提供高精准度的时间校正（LAN上与标准间差小于1毫秒，WAN上几十毫秒），且可介由加密确认的方式来防止恶毒的协议攻击。

    NTP配置原则：
  • 一级分行核心设备指向总行NTP;
  • 一级分行非核心设备指向分行核心;
  • 二级分行核心设备指向一级分行核心;
  • 二级分行其他设备指向二级分行核心。

NTP部分配置说明：

表1-9                                                     NTP配置

配置命令 作用
clock timezone UTC add 8 指定时区为东8区
ntp-service authentication enable 使能NTP认证功能
ntp-service refclock-master 3 指定本地时钟作为NTP主时钟，层次为3，为其它设备提供同步时间
ntp-service reliable authentication-keyid 1 配置认证密钥ID1是可信的
ntp-service authentication-keyid authentication-mode md5 <ntp密钥> 配置MD5密文认证密钥，ID为1，密钥为{{ NTP_KEY }}
ntp-service unicast-server <ntp主服务器>source-interface loopback 0 authentication-keyid 1 preference 一级分行核心设备指向总行NTP（主：21.14.0.100 备：21.14.0.101）
undo ntp-service server disable 使能NTP服务器功能

ntp-service reliable authentication-keyid 1
ntp-service unicast-server 21.14.0.100 source-interface loopback 0 authentication-keyid 1 preference 
ntp-service unicast-server 21.14.0.101 source-interface loopback 0 authentication-keyid 1
undo ntp-service server disable
#
分行非核心设备NTP配置

#

ntp-service authentication-keyid 1 authentication-mode md5 <ntp密钥>

ntp-service unicast-server <ntp备服务器> source-interface loopback 0 authentication-keyid 1
ntp-service server disable

1.10 办公交换机端口配置

按总行规定，办公交换机连接终端的端口需要配置以下接入认证：

  • 1、802.1x认证 ；⚠
    • 802.1x:终端支持802.1x协议，且可以安装天珣客户端软件
    • mac-auth:终端支持802.1x协议，但无法安装天珣客户端软件
  • 2、MAC地址绑定（对于不支持802.1x的终端)

办公交换机的802.1x认证服务器一般为radius认证服务器，通常在分行由天珣服务器负责802.1x的认证。

华为S系列交换机支持NAC功能，该功能主要用于用户认证和控制用户的网络访问权限采，在NAC网络中若管理员关闭了NAC认证功能，则任何用户不经过接入控制即能够访问网络。因此，为保护网络安全，建议开启NAC认证，保证用户经过认证后才可以接入网络。

关于NAC功能的统一模式与传统模式：
NAC功能支持传统配置模式以及统一配置模式。相比传统模式，统一模式具有以下优势。

  • 命令行格式设计贴近用户实际使用场景，更易理解。
  • 功能点设计去除了相似的概念，并且配置逻辑更加简单易懂。

基于上述优势，建议使用统一模式部署NAC功能。V200R005C00及以后版本默认为统一模式

⚠传统模式与统一模式相互切换后，管理员必须重启设备，新配置模式的各项功能才能生效。

可以通过display authentication mode 命令查看当前的NAC模式。

统一模式（V200R005C00-V200R008C00）端口配置部分配置说明：

表1-13                                                     统一模式（V200R005C00-V200R008C00）端口配置

配置命令 视图 作用
domain tianxun 系统视图 指定全局默认域 为tianxun
dot1x authentication-method eap 系统视图 指定配置802.1x用户的认证方式EAP,默认为CHAP
authentication single-access 系统视图 仅允许用户通过一种方式的接入认证。这样当用户通过一种方式的认证后，设备不会对其他认证方式的认证报文进行处理。
service-scheme authfail 3A视图 指定认证失败服务模板并命名为 authfail
user-vlan <逃生vlan号> 3A视图 指定逃生vlan号
authentication event authen-server-down action authorize service-scheme <authfail> response-fail 接口视图 指定服务器down时的服务策略为 authfail
authentication dot1x-mac-bypass 接口视图 使能MAC旁路认证功能
authentication dot1x mac-authen 接口视图 使能端口准入认证的方法为dot1x与mac认证
dot1x mac-bypass mac-auth-first(可选) 接口视图 对于无法安装dot1x客户端软件的服务器，可以指定优先使用MAC认证，加速认证过程

V200R005C00及以后版本默认为统一模式,对于版本较早的设备可能存在使用传统模式的情况，可以通过display authentication mode 命令查看当前的NAC模式。传统模式下无法在同一个端口同时使用多种认证方式，因此 dot1x与MAC认证只能2选1。
