1.1 设备命名配置

设备命名需采用以下统一命名规则：

AA-BB-CC-DD-EE

  • AA: 分行标识
  • BB: 功能区域代码
  • CC: 设备类型代码
  • DD: 机柜位置信息(可选)
  • EE: 设备编码

设备命名配置：

hostname XXFH-CO-CS-01

1.3 AAA配置

AAA ，认证(Authentication)：验证用户的身份与可使用的网络服务；授权(Authorization)：依据认证结果开放网络服务给用户；计帐(Accounting)：记录用户对各种网络服务的用量，并提供给计费系统。整个系统在网络管理与安全问题中十分有效。常用的AAA协议是Radius（RFC 2865，RFC 2866），TACACS（RFC1492），思科设备AAA认证主要用到的是TACACA协议。

分行核心设备指向总行3A服务器，其他设备指向分行的3A服务器。

AAA服务部分配置说明：

表1-3                                                     AAA服务配置

配置命令 作用
aaa group server tacacs+ 3a 建立tacacs服务器组，并命名为3a
server 21.14.0.122 总行主认证服务器地址21.14.0.122
server 21.14.0.123 总行备认证服务器地址21.14.0.123
ip tacacs source-interface Loopback0 指定3A认证发起的端口，此端口IP须与3A服务器上一致
tacacs-server host 21.14.0.122 key {{ FH3A_KEY }} 总行3a认证密钥为 {{ FH3A_KEY }}
tacacs-server host 21.14.0.123 key {{ FH3A_KEY }} 总行3a认证密钥为 {{ FH3A_KEY }}

1.4 SSH登陆配置

SSH是一种协议标准，其目的是实现安全远程登录以及其它安全网络服务。目前要求所有命令行登陆的设备均需使用SSH协议。

SSH部分配置说明：

表1-4                                                     SSH配置

配置命令 作用
username <本地用户> privilege 15 secret 5 <本地密码> 指定本地用户名密码，分行用户名规范“分行简称”，如北京分行用户名为 BJFH
enable secret 5 <enable密码> 使用加密分式指定enable密码
ip ssh version 2 指定SSH版本为V2
ip domain name cgbchina.com.cn 指定设备域名为cgbchia.com.cn
crypto key generate rsa general-keys modulus 2048 为加密会话生成密钥，要求RSA长度为2048
ip access-list extended VTY_ACL 分行核心设备允许下列总行地址登陆，其他非核心设备仅需允许总行网管地址登陆
permit tcp host 21.4.0.88 any 总行跳板机服务器IP
permit tcp host 21.15.255.1 any 总行核心交换机IP,作为跳板机服务器的备份
permit tcp host 21.15.255.2 any 总行核心交换机IP,作为跳板机服务器的备份
permit tcp host 10.2.95.211 any 总行网管地址
permit tcp 21.14.0.0 0.0.0.255 any 总行网管地址
permit tcp <分行源地址> any 分行地址请管理员自行添加，但源不允许为any

1.5 生成树配置

生成树协议STP(Spanning Tree Protocol)的主要功能有两个：一是在利用生成树算法、在以太网络中，创建一个以某台交换机的某个端口为根的生成树，避免环路。二是在以太网络拓扑发生变化时，通过生成树协议达到收敛保护的目的。分行使用的生成树协议有快速生成树RSTP（思科设备），与多进程生成树协议MSTP（非思科设备）

生成树部分配置说明：

表1-5                                                     生成树配置

配置命令 作用
spanning-tree mode rapid-pvst 采用快速生成树方式，比普通生成树收敛快速
spanning-tree extend system-id 采用扩展的生成树ID标识，即“生成树ID=VLAN+生成树优先级”
spanning-tree vlan <vlan-list> priority <优先级>
或者
spanning-tree vlan <vlan-list> root primary|secondary		 指定根桥与次根桥的优先级

修改生成树类型以及根节点会引起生成树重新收敛，可能导致业务流量中断，建议选择变更窗口期进行修改。

生成树配置命令如下：

主交换机，配置如下：
spanning-tree mode rapid-pvst
spanning-tree extend system-id
spanning-tree vlan <vlan-list> root primary
备交换机，配置如下：
spanning-tree mode rapid-pvst
spanning-tree extend system-id
spanning-tree vlan <vlan-list> root secondary
其它交换机，配置如下：
spanning-tree mode rapid-pvst
spanning-tree extend system-id
1.6 SNMP配置

SNMP(Simple Network Management Protocol)是一种简单网络管理协议，它属于TCP/IP五层协议中的应用层协议，用于网络管理的协议。SNMP主要用于网络设备的管理。由于SNMP协议简单可靠 ，受到了众多厂商的欢迎，成为了目前最为广泛的网管协议。

SNMP的工作方式：管理员需要向设备获取数据，所以SNMP提供了【RO】操作；管理员需要向设备执行设置操作，所以SNMP提供了【RW】操作；设备需要在重要状况改变的时候，向管理员通报事件的发生，所以SNMP提供了【Trap】操作。

SNNP部分配置说明：

表1-6                                                     SNMP配置

配置命令 作用
ip access-list standard newwg 定义允许的SNMP服务器范围
permit 10.2.95.211 总行网管地址
permit 21.14.0.0 0.0.0.255 总行网管地址
snmp-server ifindex persist 启用ifindex 持续化（接口名）。保证ifindex值在设备重启后也有效。
snmp-server community zxcv1234 RO newwg 指定SNMP的community字符，对设备的只读权限及允许访问列表
snmp-server source-interface Loopback0 指定snmp-server源端口

SNMP配置命令如下：

ip access-list standard newwg
permit 10.2.95.211
permit 21.14.0.0 0.0.0.255
！
snmp-server ifindex persist
snmp-server community zxcv1234 RO newwg
snmp-server source-interface Loopback0

1.8 NTP配置

NTP服务器【Network Time Protocol（NTP）】是用来使计算机时间同步化的一种协议，它可以使计算机对其服务器或时钟源（如石英钟，GPS等等)做同步化，它可以提供高精准度的时间校正（LAN上与标准间差小于1毫秒，WAN上几十毫秒），且可介由加密确认的方式来防止恶毒的协议攻击。

    NTP配置原则：
  • 一级分行核心设备指向总行NTP;
  • 一级分行非核心设备指向分行核心;
  • 二级分行核心设备指向一级分行核心;
  • 二级分行其他设备指向二级分行核心。

NTP部分配置说明：

表1-8                                                     NTP配置

配置命令 作用
clock timezone GMT 8 指定时区为东8区
ntp authentication-key 1 md5 <ntp密钥> 核心设备必须配置MD5，总行NTP服务器密钥 {{ NTP_KEY }}
ntp server <ntp服务器地址> key 1 prefer 总行主NTP地址21.14.0.100
ntp server <ntp服务器地址> key 1 总行主NTP地址21.14.0.101
ntp master 3 设置本地时钟作为NTP主时钟，层次为3，为其它设备提供同步时间

NTP配置命令如下：

核心网络设备NTP配置
clock timezone GMT 8
ntp authentication-key 1 md5 {{ NTP_KEY }}
ntp authenticate
ntp trusted-key 1
ntp source Loopback0
ntp update-calendar
ntp server 21.14.0.100 key 1 prefer
ntp server 21.14.0.101 key 1
ntp master 3
其它非核心网络设备配置
clock timezone GMT 8
ntp authentication-key 1 md5 <ntp密钥>
ntp authenticate
ntp trusted-key 1
ntp source Loopback0
ntp update-calendar
ntp server <ntp服务器地址> key 1 prefer
ntp server <ntp服务器地址> key 1
1.9 生产交换机端口配置

生产交换机指连接生产服务器的接入层交换机，对于生产交换机端口要求配置BPDU限制，开启风暴控制等功能。

生产交换机端口配置部分配置说明：

表1-9                                                     生产交换机端口配置

配置命令 作用
spanning-tree portfast Portfast 允许端口立即进入转发状态，防止 ESX/ESXi 主机上的链路状态变化影响 STP 拓扑。
spanning-tree bpduguard enable 收到非法的生成树 BPDU信息，将端口关闭，防止非法交换机接入导致生成树拓扑改变
storm-control broadcast level 1 开启风暴控制功能设定广播流量上限为1%
storm-control multicast level 1 开启风暴控制功能设定组播流量上限为1%
storm-control unicast level 1 开启风暴控制功能设定未知单播流量上限为1%
storm-control action log | trap 指定风暴控制动作为 log或者trap发送日志，优先指定log，无法指定log情况下才使用trap，默认动作为shutdown
snmp-server host <总行SNMP服务器地址> traps version 2c zxcv1234 storm-control 将风暴的情况发送traps到总行探针服务器，各个分行的SNMP探针服务器不相同。

1.10 办公交换机端口配置

按总行规定，办公交换机连接终端的端口需要配置以下接入认证：

  • 1、802.1x认证 ；⚠
    • 802.1x:终端支持802.1x协议，且可以安装天珣客户端软件
    • mac-auth:终端支持802.1x协议，但无法安装天珣客户端软件
  • 2、MAC地址绑定（对于不支持802.1x的终端)

办公交换机的802.1x认证服务器一般为radius认证服务器，通常在分行由天珣服务器负责802.1x的认证，办公交换机radius配置部分配置说明：

表1-10                                                     办公交换机radius配置

配置命令 作用
aaa new-model 进入aaa配置模式
aaa authentication dot1x default group radius 指定dot1x认证方式为 radius
aaa authorization network default group radius 指定dot1x授权方式为 radius
dot1x system-auth-control 全局开启dot1x认证
radius-server host <dot1x主服务器> auth-port 1812 acct-port 1813 test key <密码> 指定radius主服务器，一般为分行天徇服务器
radius-server host <dot1x备服务器> auth-port 1812 acct-port 1813 key <密码> 指定radius备服务器，一般为分行天徇服务器
radius-server deadtime 120 radius认证超时时间
radius-server dead-criteria time 10 tries 2 指定radius认证超时认证重试间隔10s,尝试次数为2

办公交换机端口配置部分配置说明：

表1-11                                                     办公交换机端口配置

特殊设备（打印机、打卡机、复印机等）不能启用802.1X，不接受天珣管理的，配置mac地址绑定（端口安全）

//*动态绑定
interface range FastEthernet0/1 - 46
 switchport port-security
 switchport port-security mac-address sticky
 switchport port-security maximum 1
 spanning-tree portfast
 spanning-tree bpduguard enable
//*静态绑定
interface range FastEthernet0/1 - 46
 switchport port-security
 switchport port-security mac-address <主机MAC地址>
 switchport port-security maximum 1
 spanning-tree portfast
 spanning-tree bpduguard enable
